# Product

## Register

product

## Users

A single technically-fluent user (the developer) on macOS, glancing at a menu bar
popover each morning to answer one question: when is power cheap today and why?
Context: just woke up, pre-coffee, seconds of attention. Secondary use: quick
follow-up questions to a local chat assistant during the day.

## Product Purpose

MorningBriefing turns Nordic electricity-market data (SE3 spot prices, nuclear
outages, live Forsmark production, Stockholm weather) into a short daily briefing
generated by a local Mistral-7B model. Success: the user knows the cheapest 4-hour
window and what is moving the price within five seconds of opening the popover.

## Brand Personality

Calm, precise, quietly premium. Native macOS 26 material (Liquid Glass), no
custom chrome. The stated direction for hero surfaces: "VERY clean and
minimalistic". Bilingual (Swedish first, English equal).

## Anti-references

- Dashboard clutter: stat-grid walls, gauges, badges competing for attention.
- Chat-bot theatrics: typing indicators everywhere, mascots, exclamation marks.
- Electron-style web chrome inside a native popover.
- Alarm-red error styling for routine data hiccups; degradation should be quiet.

## Design Principles

1. Glanceable first: the briefing sentence and the cheap window carry the screen.
2. Native material over decoration: let Liquid Glass and system typography work.
3. Degrade quietly: a failed data source dims or annotates; it never shouts.
4. Same voice in both languages: sv/en parity in every label and state.
5. Motion confirms, never performs: short springs on open, nothing looping.

## Accessibility & Inclusion

Standard macOS expectations: VoiceOver labels on icon-only controls, keyboard
access to chat input, WCAG AA contrast against the glass material, respects
system reduced-motion where feasible.
